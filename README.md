# Локальный TTS — веб-приложение для озвучки текста

Однофайловое веб-приложение (`tts.html`) на чистом HTML/CSS/JavaScript для синтеза речи **локально, без интернета и без сторонних серверов**. Использует Web Speech API → системные голоса Windows (Microsoft SAPI / Narrator).

---

## 1. Что было сломано и почему — анализ причин

В предыдущей версии не работали `rate`, `pitch`, `volume`. Реальные причины (все исправлены):

| # | Причина                                                                   | Симптом                                                  | Исправление в коде                                      |
|---|---------------------------------------------------------------------------|----------------------------------------------------------|---------------------------------------------------------|
| 1 | `keepAlive()`: `pause()`+`resume()` каждые 10 c                           | Параметры «сбрасывались» во время длинной озвучки        | Удалён. Заменён на пассивный watchdog (без pause/resume) |
| 2 | `parseFloat("")` → `NaN` при пустом инпуте                                | Chrome молча игнорирует utterance, кажется что «не работает» | `readParams()` + `clamp()` + fallback к 1.0          |
| 3 | Порядок присвоения: voice/lang устанавливались между числовыми параметрами | Chromium в ряде версий перезатирал `rate/pitch/volume`   | Сначала `lang`+`voice`, затем `rate`/`pitch`/`volume`   |
| 4 | Изменения слайдеров «на лету» — пользователь думал, что они применятся к текущему чанку | Видимая «не реакция» при движении ползунка             | Подсказка: применится со следующего фрагмента (Utterance иммутабелен) |
| 5 | `pitch` игнорируется голосами Microsoft SAPI                              | Тон вообще не меняется                                   | Баннер-предупреждение + раздел «постобработка» ниже    |
| 6 | Лимит чанка 500 симв.                                                     | Нельзя ставить длинные фрагменты                         | Расширено до 10 000 симв.                               |
| 7 | Утечки: `state.cache` без лимита, recChunks накапливаются                 | Память росла при длинных сессиях                         | LRU-кэш на 16 записей, `MediaRecorder.start(1000)`      |
| 8 | Нет валидации/ретраев                                                     | Один сбой движка останавливал всю очередь                | `onerror` → ретрай чанка 2 раза, пропуск с пометкой `err` |

## 2. Как правильно передавать параметры в Web Speech API

```js
// 1) Создаём utterance.
const u = new SpeechSynthesisUtterance(text);

// 2) СНАЧАЛА язык и голос.
u.lang  = "ru-RU";
u.voice = chosenVoice;        // объект из speechSynthesis.getVoices()

// 3) ПОТОМ числовые параметры — после валидации.
u.rate   = clamp(rate,   0.1, 10);   // 1.0 — нормальная скорость
u.pitch  = clamp(pitch,  0,   2);    // 1.0 — нормальный тон
u.volume = clamp(volume, 0,   1);    // 1.0 — максимум

// 4) Передаём в очередь движка.
speechSynthesis.speak(u);
```

**Ключевые правила:**

- Никогда не передавайте `NaN`. Парсите явно, валидируйте `Number.isFinite`.
- Параметры читаются **в момент `.speak()`**. После старта менять `u.rate` уже поздно — это особенность спецификации.
- Не вызывайте `pause()`+`resume()` без необходимости — на Chromium 100–124 это известная регрессия, частично затирающая параметры.

| Движок / API                                | rate                | pitch                | volume          | Где работает                  |
|---------------------------------------------|---------------------|----------------------|-----------------|-------------------------------|
| **Web Speech API + Microsoft SAPI (Windows)** | ✅ ограниченно (≈0.5–2.0) | ❌ обычно игнорируется | ✅                | Chrome/Edge на Windows        |
| **Web Speech API + Google voices (онлайн)** | ✅ 0.1–10           | ✅                   | ✅              | Chrome (требует сети)         |
| **Web Speech API + macOS Speech**           | ✅                  | ✅ частично          | ✅              | Safari/Chrome на macOS        |
| **Web Speech API + Android TTS**            | ✅                  | ✅                   | ⚠️ от движка    | Chrome Android                |
| **Piper (Python/C++ CLI)**                  | через `--length_scale` | ❌ напрямую нет       | через ffmpeg    | Локально, любой OS            |
| **Coqui XTTS**                              | ✅                  | ✅ (через emo/speed) | ✅              | Локально, нужен GPU/CPU       |
| **eSpeak NG**                               | ✅ (-s)             | ✅ (-p)              | ✅ (-a)         | Локально, кроссплатформенный  |
| **RHVoice**                                 | ✅                  | ✅                   | ✅              | Windows/Linux, рус/укр        |
| **pyttsx3 (Python)**                        | ✅ (rate)           | ❌ (зависит от драйвера) | ✅ (volume) | Локально через SAPI/NSSpeech  |
| **gTTS (Google Translate)**                 | ❌                  | ❌                   | через ffmpeg    | Онлайн                        |

**Главный вывод:** если важна **независимая регулировка тона** без интернета — берите eSpeak NG, RHVoice или Coqui XTTS. Голоса Microsoft SAPI на Windows игнорируют `pitch` на уровне самого движка, и веб-страница тут бессильна.

## 4. Постобработка `pitch` для движков, которые его не поддерживают

Если выбран SAPI-голос и тон не меняется — единственный рабочий способ в браузере: **сохранить аудио → сдвинуть высоту через AudioContext**.

```js
// pitch-shift через изменение playbackRate в OfflineAudioContext
async function shiftPitch(file, semitones) {
  const ratio = Math.pow(2, semitones / 12);
  const buf = await new AudioContext().decodeAudioData(await file.arrayBuffer());
  const off = new OfflineAudioContext(buf.numberOfChannels,
                                     Math.ceil(buf.duration * buf.sampleRate / ratio),
                                     buf.sampleRate);
  const src = off.createBufferSource();
  src.buffer = buf;
  src.playbackRate.value = ratio;   // меняет и темп, и высоту
  src.connect(off.destination);
  src.start();
  const rendered = await off.startRendering();
  return rendered;
}
```

Это меняет высоту вместе с темпом. Чтобы изменить высоту **независимо**, нужен time-stretch:
- библиотеки **SoundTouch.js** или **Rubberband.wasm** в браузере,
- либо `ffmpeg -af "rubberband=pitch=1.12"` оффлайн.

В UI приложения это пока не вынесено: сохраните `.webm`, обработайте сторонним инструментом.

## 5. Обработка длинных текстов (до 10 000 симв. на фрагмент)

### Алгоритм разбиения (без потери предложений)

1. Нормализация переносов `\r\n → \n`.
2. Сплит по разделителям конца предложения: `(?<=[.!?…])\s+` и `\n{2,}`.
3. Объединение последовательных предложений в фрагменты, пока их суммарная длина ≤ `chunkSize`.
4. Если одно предложение длиннее лимита — режется по последнему `,`, иначе по последнему пробелу, иначе по жёсткой границе.
5. Кэш разбиений (LRU, 16 записей) ускоряет повторное проигрывание.

### Очередь синтеза

- `state.currentChunks: string[]` — массив фрагментов.
- `playChunks()` рекурсивно вызывает следующий через `setTimeout(_, 60)` в `onend`.
- На `onerror` (кроме `canceled`/`interrupted`) — ретрай чанка до 2 раз, после чего пропуск с пометкой `err`.
- Очередь в DOM рендерится максимум на 200 элементов — остальное скрыто за плашкой «… и ещё N фрагментов».

### Защита от зависаний UI

- Никаких блокирующих циклов: всё на событиях `onstart`/`onend` и `setTimeout`.
- Пересчёт статистики на ввод текста — через **дебаунс 150 мс**.
- Для текста > 200 000 симв. `recomputeStats()` использует приближённую оценку количества чанков, чтобы не упираться в сплиттер на каждое нажатие.
- Жёсткий потолок 1 000 000 символов в `validateBeforePlay()`.
- Watchdog 1.5 с проверяет, что движок отвечает: если `speaking=false` > 3 с при активном плеере — рестартует текущий чанк.

### Оптимизация памяти

- `state.cache` — LRU-кэш с лимитом 16, эвакуация старейшей записи.
- `state.utter = null` сразу после `onend` — освобождает Utterance для GC.
- `MediaRecorder.start(1000)` — данные сбрасываются раз в секунду, а не копятся в одном Blob до конца.
- На `beforeunload` все потоки и рекордеры останавливаются.

## 6. Запуск

```powershell
# 1. Самый простой способ — открыть напрямую
start tts.html

# 2. Через локальный сервер (нужно для записи аудио в HTTPS-контексте на некоторых браузерах)
python -m http.server 8080
# http://localhost:8080/tts.html
```

### Браузеры
| Браузер           | TTS | Запись  |
|-------------------|-----|---------|
| Chrome / Edge     | ✅  | ✅      |
| Firefox           | ✅  | ⚠️ ограниченно |
| Safari            | ✅  | ⚠️      |

### Зависимости
**Нулевые.** Только встроенные браузерные API: `SpeechSynthesis`, `MediaRecorder`, `getDisplayMedia`, `AudioContext`.
