# Локальный TTS — веб-приложение для озвучки текста

Однофайловое веб-приложение (`tts.html`) на чистом HTML/CSS/JavaScript для синтеза речи **локально, без интернета и без сторонних серверов**. Использует Web Speech API, который под капотом обращается к системным голосам Windows (Microsoft SAPI / Narrator voices).

---

## 1. Архитектура

```
┌─────────────────────────────────────────────────────────────┐
│                          tts.html                           │
│                                                             │
│  ┌──────────────┐    ┌───────────────┐   ┌──────────────┐   │
│  │  UI-слой     │ →  │ Контроллер    │ → │ Очередь      │   │
│  │  (DOM, CSS)  │    │ playback      │   │ фрагментов   │   │
│  └──────────────┘    └───────┬───────┘   └──────┬───────┘   │
│                              │                  │           │
│                              ▼                  ▼           │
│                    ┌─────────────────┐  ┌────────────────┐  │
│                    │ SpeechSynthesis │  │  Cache (Map)   │  │
│                    │   (Web Speech)  │  │ ключ→chunks    │  │
│                    └────────┬────────┘  └────────────────┘  │
│                             │                               │
│                             ▼                               │
│                  ┌──────────────────────┐                   │
│                  │   Windows SAPI 5     │                   │
│                  │  Microsoft Voices    │                   │
│                  └──────────────────────┘                   │
│                                                             │
│  Дополнительно: MediaRecorder + getDisplayMedia(audio)      │
│  для сохранения озвучки в `.webm`-файл.                     │
└─────────────────────────────────────────────────────────────┘
```

Ключевые принципы:

- **UI ↔ движок развязаны через события** `onstart` / `onend` / `onerror` каждого `SpeechSynthesisUtterance` — нет ни одного блокирующего вызова в основном потоке.
- **Очередь фрагментов** заменяет один длинный Utterance: длинные тексты в Chromium-движках Windows обрываются на ~200–250 символах, поэтому мы режем текст и проигрываем подряд.
- **Кэш разбиений** (`Map`) хранит результаты `splitText()`, чтобы повторное нажатие «Озвучить» на том же тексте не пересчитывало деление.
- **Keep-alive таймер** периодически делает `pause+resume`, обходя баг Chrome, при котором синтез «засыпает» после ~15 c.
- **Запись звука** реализована через `getDisplayMedia({audio:true})` + `MediaRecorder`, т.к. Web Speech API **не отдаёт сырое аудио**.

## 3. Реализация ключевых функций

| Возможность              | Где в коде (`tts.html`)                           | Кратко                                                                 |
|--------------------------|---------------------------------------------------|------------------------------------------------------------------------|
| Поле ввода текста        | `<textarea id="text">`                            | Авто-пересчёт статистики при `input`.                                  |
| Кнопка «Озвучить»        | `startPlaybackInternal()`                         | Делит текст, заполняет очередь, запускает `playChunks()`.              |
| Пауза / Продолжить       | `pause()` / `resume()`                            | `speechSynthesis.pause()` / `.resume()`.                               |
| Остановка                | `stop()`                                          | `speechSynthesis.cancel()` + сброс состояния.                          |
| Скорость / Тон / Громкость | поля `rate`/`pitch`/`vol`, применяются в `playChunks()` | `utter.rate`, `utter.pitch`, `utter.volume`.                     |
| Выбор языка / голоса     | `loadVoices()` + `fillVoiceSelect()`              | Группировка `getVoices()` по `lang`, селект по умолчанию — `ru-*`.     |
| Сохранение в файл        | `startRecordingAndPlay()` / `stopRecording()`     | Захват аудио вкладки → `MediaRecorder` → скачивание `.webm`.           |
| Длинные тексты           | `splitText()`                                     | Разбиение по предложениям и затем по запятым/пробелам ≤ chunkSize.     |
| Очередь                  | `state.currentChunks`, `playChunks()`             | Каждый чанк = свой Utterance, цепочка через `onend`.                   |
| Кэш разбиений            | `state.cache: Map`                                | Ключ: `chunkSize::length::prefix(64)`.                                 |
| Защита от блокировки UI  | event-driven архитектура + `setTimeout(playChunks, 60)` | Никаких `while`/`await blocking`.                                      |
| Keep-alive               | `setInterval(keepAlive, 10000)`                   | Обход «засыпания» Chrome.                                              |

## 4. Запуск

### Самый простой способ
1. Скачайте файл `tts.html`.
2. Дважды кликните по нему — откроется в браузере.
3. Готово.

### Через локальный HTTP-сервер (рекомендуется для записи аудио)
Некоторые функции, особенно `getDisplayMedia`, могут требовать «secure context»:

```powershell
# Python 3 (встроен в Windows 10/11 при установленном Python)
cd C:\path\to\project
python -m http.server 8080
# затем: http://localhost:8080/tts.html
```

Либо:

```powershell
# Node.js
npx serve .
```

| Chrome / Edge     | ✅  | ✅            |
| Firefox           | ✅  | ⚠️ ограниченно |

## 5. Зависимости

**Нулевая**. Только встроенные браузерные API:

- [`SpeechSynthesis`](https://developer.mozilla.org/docs/Web/API/SpeechSynthesis) — синтез речи.
- [`MediaRecorder`](https://developer.mozilla.org/docs/Web/API/MediaRecorder) — запись.
- [`getDisplayMedia`](https://developer.mozilla.org/docs/Web/API/MediaDevices/getDisplayMedia) — захват звука вкладки.

Системные требования (Windows):

- Windows 10 / 11.
- Установлены пакеты речи: *Параметры → Время и язык → Речь → Управление голосами*. Для русского — Microsoft Irina (десктоп) или Microsoft Pavel/Maxim/Tatyana.

## 6. Производительность и масштабирование

- **Не блокирует UI**: всё через события и `setTimeout`. Длинный текст в 100 000 символов разбивается за ≤ 20 мс, остальное обрабатывает аудиодвижок.
- **Кэш разбиений** убирает повторную работу при повторном проигрывании.
- **Микро-задержка 60 мс** между чанками (`setTimeout(playChunks, 60)`) даёт движку «выдохнуть» и предотвращает гонки событий в Chromium.
- **Keep-alive** убирает практически все случаи «зависания» Chrome на длинных сессиях.

### Если нужен серверный масштаб

Для бизнес-сценариев (рассылка озвучек, поточная генерация) Web Speech не подходит — он привязан к клиентской машине. Рекомендуемые офлайн-движки уровня нейросетей:

| Движок    | Качество  | Языки     | Лицензия | Использование                                |
|-----------|-----------|-----------|----------|----------------------------------------------|
| **Piper** | Высокое   | 40+ (рус) | MIT      | CLI, gRPC; быстрый, без GPU                  |
| Coqui XTTS| Очень высокое | 16+    | MPL-2.0 | Клонирование голоса; нужен GPU               |
| RHVoice   | Среднее   | рус, укр  | LGPL     | Лёгкий, встроенные движки для Windows        |
| eSpeak NG | Низкое    | 100+      | GPL      | Минимальный размер, для быстрых заглушек     |

Архитектура расширения: добавить в `tts.html` опцию «движок», и если выбран не «локальный браузер», слать `fetch('/api/tts', {body:{text, voice}})` на сервер с Piper, получая `audio/wav`.

## 7. Качество голоса — практические советы

1. **Установите дополнительные голоса**: Windows-Eyes, Microsoft Irina, RHVoice Anna. Они подхватятся через `getVoices()` автоматически.
2. **Тон `pitch`** в диапазоне 0.9–1.1 звучит естественно; крайние значения дают «робота».
3. **Скорость `rate` 0.95–1.05** — большинство голосов SAPI деградирует при `rate > 1.4`.
4. **Подготовка текста**: расставьте запятые и точки явно, разверните аббревиатуры («гг.» → «годов»), числа лучше прописью.
5. **Размер фрагмента 150–250 симв.** — компромисс между плавностью и стабильностью движка.

## 8. Возможные расширения

- SSML-разметка (паузы, ударения) — поддерживается частично, зависит от голоса.
- Подсветка текущего слова через событие `onboundary` (`utter.onboundary = ...`).
- История озвучек в `localStorage`.
- Drag-and-drop `.txt`/`.docx` — `FileReader` + парсер.
- Тёмная/светлая тема (сейчас тёмная, переменные CSS уже вынесены в `:root`).
- PWA-манифест — приложение можно «установить» как desktop.
